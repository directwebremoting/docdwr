Commercial Support

We can provide a number of support packages to help you with Ajax/DWR development.

The three primary benefits of our support packages include:

  • Unlimited advice for your designated team members for questions about best practices, approaches, etc.
  • A set number of hours dedicated to fixing problems you have either with DWR itself, or your use of DWR.
  • priority support to get a solution to your problem right away.

For information about commercial support, please . If you have questions about DWR please don't send them to the authors directly. There are far more people that can help on the mailing lists, so you'll get a faster response that way.

DWR-Security

The idea behind the dwr-security mailing list is that we should have a way to discuss DWR security while minimizing the dangers associated with wide public dissemination. The list is not about keeping problems secret forever, but it is about responsible disclosure.

Just to be clear: dwr-security is NOT another option for getting support when something doesn't work.

Membership rules: Since dwr-security is about responsible disclosure, we would like to know who the members are. Almost everyone wants to subscribe from a gmail or similar web-based address. That's OK, but we want to link web addresses to some type of corporate address OR have some other way of positively identifying you.

  • If you are subscribing using a corporate email address, please make sure we know your full name and job title in the request.
  • If you are subscribing using a web-based email address, please apply in the same way but also send an email from your corporate address to [joe at getahead dot org] at the same time.
  • If you don't have a corporate email address, please find some other way to identify yourself to us.

Applications to join can be made using Google Groups.

Anyone can post to the list using the address: [dwr-security at googlegroups.com]. You don't need to be a member to post to the list.
